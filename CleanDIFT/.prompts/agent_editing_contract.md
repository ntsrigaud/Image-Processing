AGENT EDITING CONTRACT (MANDATORY)

You are operating under a strict file-editing contract.

Rules:

1. You MUST NOT claim that a file has been edited unless the edit is verifiably real.

2. Every claimed file edit MUST be accompanied by ONE of the following:
   a) A unified diff (git-style) showing the exact before/after changes, OR
   b) The full updated content of the modified section, copied verbatim as it appears in the file.

3. If neither a diff nor verbatim updated content is shown, the edit is assumed NOT to have occurred.

4. You MUST explicitly list:
   - The exact file path(s) edited
   - The type of change (added, modified, removed)
   - The scope of the change (e.g., “Markdown cell 12”, “function evaluate_pck”, “table in RESULTS.md”)

5. For Jupyter notebooks (.ipynb):
   - You MUST specify which cell indices were modified.
   - You MUST paste the full updated Markdown or code cell content.
   - You MUST NOT claim notebook execution or output changes unless explicitly requested.

6. You MUST NOT summarize edits in place of showing them.
   Descriptions without evidence are considered invalid.

7. If a file cannot be edited due to environment limitations:
   - You MUST say so explicitly.
   - You MUST fall back to generating a unified patch or full replacement content.

8. If an edit fails, partially applies, or is uncertain:
   - You MUST report the failure.
   - You MUST NOT imply success.

9. Silence or omission is NOT confirmation of success.

Compliance with this contract is required before proceeding to any next task.
